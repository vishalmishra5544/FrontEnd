Structural Document & Metadata

Foundational tags that declare the document type and provide metadata for browsers, crawlers, and tooling. Most live inside <head>.

Core & Head Elements
  • <!DOCTYPE> Declares HTML version to the browser (HTML5). Required at the top of the document. 
    <!DOCTYPE html>
  • <html> Root element wrapping the entire document. Use lang and (optionally) dir for accessibility and i18n. 
    <html lang="en"> ... </html>
  • <head> Container for metadata: <title>, <meta>, styles, scripts, and links. 
    <head> ... </head>
  • <title> Document title shown in the browser tab and used by search engines. 
    <title>Basics of HTML</title>
  • <base>optional Sets the base URL and/or default target for all relative links in the page. 
    <base href="https://example.com/" target="_blank">
  • <meta> Encodes metadata (charset, viewport, SEO). Critical for mobile responsiveness and correct encoding. 
    <meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="all basics of html">
  • <link> Links external resources like CSS, icons, and preconnect hints. 
    <link rel="stylesheet" href="styles.css">
<link rel="icon" href="/favicon.ico">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  • <style> Embeds CSS directly in the document head. Prefer external files for large stylesheets. 
    <style> body { font-family: system-ui; } </style>
  • <script> Loads/executes JavaScript. Use defer for scripts that touch the DOM; type="module" for ESM. 
    <script src="app.js" defer></script>
<script type="module"> import init from './main.js'; init(); </script>
  • <noscript> Fallback content for users with JavaScript disabled or blocked. 
    <noscript>Please enable JavaScript for the best experience.</noscript>

Semantic Layout

High-level structural elements that improve accessibility, SEO, and maintainability.

Page Sections
  • <body>Container for all visible content of the page.
    <body> ... visible content ... </body>
  • <header>Introductory content: logo, title, primary nav; can appear within page or within an article/section.
    <header>...site nav...</header>
  • <nav>Navigation block containing links to major sections of the site or page.
    <nav><a href="#media">Media</a></nav>
  • <main>Primary content unique to the document. Only one per page.
    <main>...main content...</main>
  • <section>Thematically grouped content, typically with a heading.
    <section aria-labelledby="s1"><h2 id="s1">Overview</h2>...</section>
  • <article>Self-contained composition (blog post, card, comment) that could stand alone.
    <article><h3>Post</h3><p>...</p></article>
  • <aside>Tangential or complementary content (sidebars, callouts, ads).
    <aside>Related links</aside>
  • <footer>Footer for its nearest sectioning root (page or article). Often contains legal, contacts, or nav.
    <footer>© 2025 Vishal</footer>
  • <address>Contact info for a person/organization related to its nearest ancestor <article> or the document.
    <address>Email: hello@example.com</address>

Text & Typography

Headings, paragraphs, inline semantics, and editorial emphasis that convey structure and meaning.

Headings & Blocks
  • <h1> … <h6>Six levels of headings, from most important (<h1>) to least (<h6>).
    <h1>Main Title</h1>
  • <p>Paragraph of text; the most common block element.
    <p>This is a paragraph.</p>
  • <pre>Preformatted text that preserves whitespace and line breaks.
    <pre>line 1
line 2</pre>
  • <blockquote>Extended quotation, optionally with a cite URL.
    <blockquote cite="https://example.com">Quote</blockquote>
  • <hr>Thematic break between paragraphs/sections.
    <hr>
Inline Semantics & Edits
  • <strong>Strong importance; typically bold. Conveys meaning to assistive tech.
    <strong>Important</strong>
  • <em>Emphasis; typically italic. Indicates stress emphasis in speech.
    <em>Emphasized</em>
  • <b>Stylistic offset without semantic importance.
    <b>Bold look</b>
  • <i>Alternate voice/mood (terms, idioms, thoughts), not emphasis.
    <i>Latin phrase</i>
  • <small>Side comments, disclaimers, legal text.
    <small>T&C apply</small>
  • <u>Stylized underline (not for links). Avoid for link-like styling.
    <u>Underlined</u>
  • <s>Content no longer accurate/relevant (rendered struck through).
    <s>Old price</s>
  • <ins>Inserted text (diff/edits). Can carry cite and datetime.
    <ins>Added</ins>
  • <del>Deleted text (diff/edits). Can carry cite and datetime.
    <del>Removed</del>
  • <sup>Superscript (math/footnotes).
    X<sup>2</sup>
  • <sub>Subscript (chemical formulas).
    H<sub>2</sub>O
  • <br>Line break inside text (use sparingly; prefer CSS for spacing).
    Hello<br>World
  • <wbr>Optional word break opportunity for long tokens/URLs.
    supercalifragilisticexpiali<wbr>docious
  • <span>Generic inline container for styling or scripting hooks.
    <span class="tag">chip</span>
  • <code>Inline code fragment; pair with <pre> for blocks.
    <code>const x=1</code>
  • <kbd>User input via keyboard/voice.
    <kbd>Ctrl</kbd>+<kbd>C</kbd>
  • <samp>Sample output from a program or system.
    <samp>404 Not Found</samp>
  • <var>Variable in a mathematical or programming context.
    <var>x</var> = 42
  • <mark>Highlight text as if marked with a highlighter pen.
    <mark>search term</mark>
  • <abbr>Abbreviation/acronym; use title to expand.
    <abbr title="HyperText Markup Language">HTML</abbr>
  • <dfn>Term being defined; often the first occurrence.
    <dfn>API</dfn> — Application Programming Interface
  • <cite>Reference to a creative work (book, site, paper).
    <cite>MDN Web Docs</cite>
  • <q>Inline quotation; user agents may add quotes automatically.
    <q>Short quote</q>
  • <time>Machine-readable dates/times using datetime.
    <time datetime="2025-08-24">Aug 24, 2025</time>
  • <ruby>, <rt>, <rp>Ruby annotations for East Asian typography; <rp> adds fallback parentheses.
    <ruby>漢<rt>kan</rt><rp>(</rp><rp>)</rp></ruby>
  • <data>Associates a human-readable value with a machine-readable one via value.
    <data value="101">Product #101</data>
Text Direction
  • <bdi>Isolates a span of text that might have a different directionality.
    <bdi>علي</bdi>
  • <bdo>Overrides the current text direction with dir.
    <bdo dir="rtl">abc</bdo>

Links & Anchors

Create hyperlinks and image maps to navigate within and across documents.

Anchors & Maps
  • <a>Hyperlink to URLs, fragments, downloads, or actions.
    <a href="#forms">Jump to Forms</a>
  • <map> & <area>Clickable regions on images using coordinates.
    <img src="plan.png" usemap="#m">
<map name="m">
  <area shape="rect" coords="0,0,100,100" href="/A">
</map>

Lists

Ordered, unordered, and description lists to group related items.

List Elements
  • <ul>, <ol>, <li>Unordered and ordered lists with list items.
    <ol><li>Step 1</li><li>Step 2</li></ol>
  • <dl>, <dt>, <dd>Description lists with term and definition pairs.
    <dl><dt>HTML</dt><dd>Markup language</dd></dl>

Tables

Represent tabular data with semantic grouping for headers, bodies, and footers.

Table Elements
  • <table>Table container. Use semantic sections for accessibility.
    <table>...</table>
  • <caption>Title for the table; improves accessibility.
    <caption>Sales 2025</caption>
  • <thead>, <tbody>, <tfoot>Semantic grouping of header, body, and footer rows.
    <thead><tr><th>Q1</th></tr></thead>
  • <tr>, <th>, <td>Rows and header/data cells; set scopes on <th>.
    <tr><th scope="col">Name</th><td>Vishal</td></tr>
  • <colgroup>, <col>Define column groups and per-column attributes (e.g., width).
    <colgroup><col span="2" style="width:50%"></colgroup>

Forms

Collect, validate, and submit user input with semantic controls and labels.

Form Elements
  • <form>Form container with action/method. Use autocomplete and novalidate as needed.
    <form action="/signup" method="post">...</form>
  • <label>Associates text with a control via for (or by wrapping).
    <label for="email">Email</label>
  • <input>Versatile control: text, email, password, number, date, file, etc.
    <input id="email" type="email" required placeholder="you@example.com">
  • <textarea>Multiline text input; use rows/cols or CSS for sizing.
    <textarea rows="3"></textarea>
  • <select>, <option>, <optgroup>Dropdown menus; group related options with labels.
    <select><optgroup label="A"><option>Alpha</option></optgroup></select>
  • <button>Clickable button; default type is submit inside forms.
    <button type="button">Click</button>
  • <fieldset> & <legend>Group related controls with a caption for screen readers.
    <fieldset><legend>Profile</legend>...</fieldset>
  • <datalist>Provides suggested values for an <input list> field (autocomplete).
    <input list="cities">
<datalist id="cities"><option value="Delhi"></datalist>
  • <output>Displays calculation results tied to form controls.
    <output for="a b">10</output>
  • <progress>Indicates task progress with value and max.
    <progress value="30" max="100">30%</progress>
  • <meter>Displays a scalar measurement within a known range.
    <meter value="0.7" min="0" max="1">70%</meter>

Media & Embeds

Embed images, audio/video, vector graphics, plugins, and external documents.

Images & Figures
  • <img>Inline bitmap image. Always include alt for accessibility.
    <img src="hero.jpg" alt="Scenic view" loading="lazy">
  • <picture> & <source>Art-direction and modern formats via multiple sources with media/type queries.
    <picture>
  <source srcset="img.avif" type="image/avif">
  <img src="img.jpg" alt="...">
</picture>
  • <figure> & <figcaption>Self-contained media with an accessible caption.
    <figure><img src="a.jpg" alt="A"><figcaption>Image A</figcaption></figure>
Audio & Video
  • <audio> & <video>Built-in media players; combine with <source> and tracks.
    <video controls width="480">
  <source src="movie.mp4" type="video/mp4">
</video>
  • <track>Timed text tracks for media (subtitles, captions). Use kind and srclang.
    <track kind="subtitles" srclang="en" src="movie.vtt" default>
Embeds & Graphics
  • <iframe>Embeds another HTML page. Set title and restrictive sandbox for security.
    <iframe src="https://example.com" title="Example" loading="lazy"></iframe>
  • <embed>Embeds external content (PDFs, plugins). Limited control vs <object>.
    <embed src="file.pdf" type="application/pdf">
  • <object> & <param>Generic embedded object with parameters for configuration.
    <object data="movie.swf">
  <param name="autoplay" value="true">
</object>
  • <canvas>Scriptable bitmap surface for 2D/3D drawings (via JS or WebGL).
    <canvas id="c" width="300" height="150"></canvas>
  • <svg>Scalable Vector Graphics markup for resolution-independent shapes and icons.
    <svg width="24" height="24" viewBox="0 0 24 24"><circle cx="12" cy="12" r="10"/></svg>
  • <math>Embeds MathML for mathematical notation (browser support varies).
    <math>...MathML...</math>

Interactive Elements

Built-in interactive widgets that work without JavaScript (but can be enhanced).

Disclosures & Dialogs
  • <details> & <summary>Disclosure widget users can expand/collapse.
    <details><summary>More</summary>Hidden text</details>
  • <dialog>Native modal/light-dismiss dialog with scripting API.
    <dialog open>Hello</dialog>

Web Components

Templates and slots for componentizing markup; typically used with Shadow DOM (JS).

Templating & Slots
  • <template>Declarative HTML that is inert until cloned via JS.
    <template id="tpl"><li>Item</li></template>
  • <slot>Placeholder inside Shadow DOM where light DOM is projected.
    <slot name="title">Fallback</slot>

Misc Utility

Helpful odds and ends that don’t fit other buckets.

Odds & Ends
  • <address>Contact information related to the nearest <article> or document.
    <address>By Vishal — Mumbai, IN</address>
  • <noscript>Displays fallback content if JS is unavailable (listed above in metadata too).
    <noscript>Please enable JS</noscript>

Deprecated / Obsolete / Non-standard

Avoid these in modern HTML5. They’re obsolete, replaced by CSS/JS, or never standardized.

Do Not Use
  • <acronym>Deprecated — use <abbr> with a title instead.
    <abbr title="World Health Organization">WHO</abbr>
  • <applet>Deprecated — use <object> or modern web APIs.
    <object data="..."></object>
  • <basefont>, <font>, <big>, <tt>Deprecated — use CSS for styling text.
    <span style="font-size:larger">...</span>
  • <center>Deprecated — center with CSS.
    <div style="text-align:center">...</div>
  • <dir>Deprecated — use <ul>.
    <ul><li>...</li></ul>
  • <frame>, <frameset>, <noframes>Deprecated — use <iframe> and CSS layout.
    <iframe src="... "></iframe>
  • <strike>Deprecated — use <s> or <del> depending on meaning.
    <s>Outdated</s>
  • <isindex>Deprecated — use proper <form> controls.
    <form><input type="search"></form>
  • <keygen>Deprecated — use Web Crypto / server-side generation.
    // use modern APIs
  • <marquee>Non-standard/obsolete — replicate with CSS animations.
    @keyframes slide{}
  • <xmp>, <plaintext>, <blink>, <spacer>Obsolete/non-standard historical tags.
